# Specification: core_vm_20260302

## Objective
Implement a core bytecode virtual machine for a subset of Scheme R5RS, providing the foundation for a spec-compliant, embeddable interpreter.

## Scope
- **Data Types**: Fixnums, Booleans, Pairs (Lists), Symbols, Procedures, Continuations.
- **Language Features**:
  - Proper tail recursion (as required by R5RS).
  - First-class continuations (`call-with-current-continuation`).
  - Lexical scoping and recursion.
- **Architectural Components**:
  - **Bytecode Compiler**: Transforms Scheme S-expressions into virtual machine bytecode.
  - **VM Interpreter**: A stack-based execution environment for the generated bytecode.
  - **Memory Management**: Custom garbage collector (Mark-and-Sweep) for automatic life cycle management of Scheme objects.
  - **REPL**: Interactive Read-Eval-Print Loop with basic ANSI coloring.
  - **File Loader**: Mechanism to load and execute Scheme scripts from the filesystem.

## Non-functional Requirements
- **Language**: Vanilla C99 (POSIX-compliant).
- **Design**: Modular, self-contained, and easily embeddable into other C applications.
- **Priorities**: Code readability and simplicity (KISS principle) over extreme performance.

## Acceptance Criteria
- Successfully compile and run core Scheme programs involving lists and recursion.
- Verify that `call/cc` behaves as expected (continuations can be captured and re-invoked).
- Ensure tail call optimization prevents stack overflow for deep iterative processes.
- Achieve >80% test coverage using the Unity framework.
- No memory leaks detected by Valgrind for standard Scheme operations.
